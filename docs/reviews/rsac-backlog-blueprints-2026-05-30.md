# rsac backlog — research + architecture blueprints (2026-05-30, Tavily/Exa/DeepWiki)

## rsac backlog consolidation — execution plan (Waves B–E)

### Scope
Seven researched/blueprinted hard-item domains plus the simpler items they reference, consolidated into four dependency-ordered execution waves. The branch already carries Wave-1/Wave-2 bridge/api/capabilities P0 fixes and ADRs 0001–0009; this plan covers the remaining safety, API-stability, CI, and perf work toward a 0.3.0 cut.

### Key research takeaways
- **macOS watch UAF (H1)** has no race-free one-wave fix: CoreAudio's PROC-based listener gives no guarantee an in-flight `watch_listener_proc` has finished when `AudioObjectRemovePropertyListener` returns. The recommended **minimal sound stopgap** is to *intentionally leak* the `WatchListenerContext` (`Box::into_raw`, never reclaimed on the success/spawn-failure paths) and stop delivery by dropping only the cloned `SyncSender` (channel-disconnect). The proper fix (block + self-owned serial dispatch queue, Chromium/Itsuki pattern) is deferred to a tracked `deferred-review` issue. The overstated doc comments at `coreaudio.rs:775-779/825-828` and ADR-0005 §5 must be rewritten to state the *actual* (leak-based) guarantee — required by the project's honesty policy.
- **Go cgo UAF (#28)** needs a **two-layer fix**: a Go-side drain-before-free barrier (`sync.WaitGroup` + `closing atomic.Bool`, the `database/sql` Rows.Close pattern) PLUS a new best-effort FFI `rsac_capture_request_stop` unblock primitive (the `sqlite3_interrupt` pattern) so `Close()` returns promptly instead of waiting out the 1s `pop_blocking` timeout. `runtime.Pinner` is the wrong tool (capture is Rust-owned, not GC-managed); holding `c.mu` across the blocking read deadlocks (the documented #28 reason). Narrowing the FFI read path from `*mut`→`*const` (i.e. `read_buffer`/`read_buffer_blocking` from `&mut self`→`&self`) also removes a separate `&mut`-aliasing UB and is C-ABI-compatible.
- **non_exhaustive (semver, AEG-1/AEG-2)** should be a single batched 0.2.x→0.3.0 boundary change: mark `AudioError`, `CaptureTarget`, `AudioSourceKind`, `PermissionStatus` `#[non_exhaustive]`; keep `SampleFormat`/`DeviceKind`/`ErrorKind`/`Recoverability` deliberately closed with a one-line doc guarantee. The attribute is a no-op in-crate (exhaustive `recoverability()`/`user_message()` keep compiling and keep forcing classification — do **not** add a `_` arm there), but the three binding crates break and need wildcard arms. The original finding **missed** three cross-crate breaks: `map_rsac_error` (rsac-ffi), `audio_error_to_pyerr` (rsac-python), and a CaptureTarget match — all need trailing arms.
- **Producer-terminal-signal (FH-1)** must be unified: every backend drives `BridgeShared` to a terminal/ending state on producer stop/death (today only Windows does, so Linux/macOS readers hang). Add a fatal sibling `signal_error()` (Running→Error) to the existing graceful `signal_done()` (Running→Stopping); hook Linux via a `.state_changed` arm (Error/Unconnected) + loop-teardown, macOS via `stop_audio_unit()` + optional `DeviceIsAlive` listener. Warrants **ADR-0010** (next free number — confirmed max is 0009).
- **async-waker + terminal binding** splits into Part A (core-only waker-contract enforcement: fail-fast in `audio_data_stream()` + self-wake fallback in `poll_next` so a non-waking backend never parks/hangs) and Part B (per-binding terminal-error delivery — notably the **inverted** Python `__next__` mapping that raises StreamError on natural end and swallows recoverable hiccups).
- **CI/build hygiene** is six independent low-risk fixes (narrow Windows watch over-skip, run enumeration_matrix on real hardware, broaden the module-DAG guard + self-test, gate macOS tap presence asserts, fix the bare `--no-default-features` build E0432, kill the cbindgen double-prefix + README clobber + add header-drift CI).
- **Perf + sinks** is five independently-shippable changes ordered low-risk-first: PU-7 (WASAPI bulk `slice::align_to::<f32>()` — **not** bytemuck, which panics on misalignment), PU-5 (always-present Condvar wake — Windows producer + every state transition only; **never** from the Linux/macOS RT callbacks, per ADR-0001; keep the bounded poll as a degrade-not-hang backstop), AEG-4/FH-6 (`drain_to` + rewire the dead 4-type sink abstraction through `record_to_file`), FH-3 (WavFileSink format validation), AEG-7/DF-04+PU-2/DF-01 (sizing/timestamps).

### Overall sequencing rationale
Wave B (safety) lands first because two of its items create the foundations later waves build on: `signal_error()`/`signal_done()` in `ring_buffer.rs` (producer-terminal-signal) is required by both Wave C's terminal-error delivery and Wave E's PU-5 wake-on-stop, and the Go barrier work establishes the FFI `request_stop` primitive. Wave C is the breaking-API window (do the single 0.3.0 churn here). Wave D is pure CI/build hygiene with no library-runtime coupling (can run nearly anytime, scheduled after the safety/API churn settles to avoid CI baseline thrash). Wave E (perf/sinks) lands last and is gated on Wave B's `ring_buffer.rs` changes to avoid serialized merge conflicts on the same hot file.

### producer-terminal-signal

**Approach:** UNIFY the producer-terminal-signal contract: every backend MUST drive BridgeShared into a terminal/ending state when its producer stops (gracefully or fatally), mirroring Windows. Distinguish graceful end (Running->Stopping via existing signal_done()) from fatal death (Running->Error). Concretely:\n\n0) BRIDGE — add a fatal sibling to signal_done() (src/bridge/ring_buffer.rs near 726). Add `pub fn signal_error(&self)` that does `self.shared.state.force_set(StreamState::Error)` + (cfg async) `self.shared.waker.wake()` — factoring the existing on_push_panic() body.
